# Правила обзвона кандидатов

VoiceScreen звонит кандидатам автоматически, но в чётких рамках. Эти правила
гарантируют, что мы не будим людей ночью и не долбим их бесконечно.

## Когда звоним

**Окно: 9:00 – 21:00 по московскому времени.**

- Если HR нажимает «Запустить обзвон» **внутри окна** — звонки уходят сразу,
  параллельно по нескольким линиям Voximplant.
- Если HR нажимает кнопку **вне окна** (например, ночью или рано утром) —
  кандидаты в очередь ставятся, но **первый звонок отложится до 9:00 утра**
  ближайшего рабочего дня. В диалоге подтверждения отображается точное время
  начала: «Старт обзвона: 28.04.2026, 09:00».

Окно одинаковое для всех кандидатов независимо от их региона. Локальные
таймзоны кандидатов будем учитывать после пилота.

## Сколько попыток

**На каждого кандидата — до 3 попыток.**

Попытка считается израсходованной независимо от исхода:

- **«Дозвонились и провели скрининг»** — попытка зачтена, проводится оценка
  по сценарию и формируется решение (`pass` / `review` / `reject`). Дальше
  кандидата не дёргаем.
- **«Не дозвонились»** (отказ от вызова, занято, не отвечает, недоступен) —
  попытка тоже зачтена, и автоматически планируется следующая попытка.

После третьей неудачной попытки кандидат помечается как
**«не дозвонились»** (`status='exhausted'`). Bulk-обзвон такого кандидата
больше не подбирает; HR может решить вручную (например, передать его
оператору).

## Промежутки между попытками при недозвоне

Если кандидат не взял трубку, мы **не звоним ему ещё раз через минуту** — это
выглядит навязчиво. Используется растущий интервал:

| Попытка | Когда                                               |
| ------- | --------------------------------------------------- |
| 1       | Сразу после нажатия «Запустить обзвон» (или в 9:00) |
| 2       | Через **30 минут** после первого недозвона          |
| 3       | Через **2 часа** после второго недозвона            |
| —       | После третьего — кандидат закрывается               |

Если рассчитанное время следующей попытки попадает за пределы окна
(например, недозвон случился в 20:45, плюс 30 минут = 21:15) — попытка
сдвигается на 9:00 следующего утра.

## Что видит HR в кабинете

В карточке кандидата:

- **Статус**: `pending` (в очереди), `in_progress` (звоним прямо сейчас),
  `exhausted` (не удалось дозвониться за 3 попытки), `done` (получили
  финальное решение).
- **Счётчик попыток** — сколько раз уже звонили.
- **Время следующей попытки** — если запланирован retry.

В таблице звонков для каждой попытки виден отдельный `Call` с собственным
`decision`. Недозвоны помечаются как `decision = not_reached` и не учитываются
в скоринге, только в счётчике.

## Регуляторика

- Первая фраза агента в каждом звонке содержит уведомление о записи
  разговора — соблюдается 152-ФЗ.
- Кандидат может в любой момент сказать «нет» / «не согласен» — звонок
  завершается прощальной фразой, оценка не проводится.

## Что можно настроить

Глобальные параметры (общие для всех клиентов; меняются разработчиком в
`.env`):

| Параметр                   | Значение по умолчанию      |
| -------------------------- | -------------------------- |
| `CALL_TIMEZONE`            | `Europe/Moscow`            |
| `CALL_WINDOW_START_HOUR`   | `9`                        |
| `CALL_WINDOW_END_HOUR`     | `21`                       |
| `CALL_MAX_ATTEMPTS`        | `3`                        |
| `CALL_RETRY_BACKOFF_MINUTES` | `[30, 120, 360]`         |

Под каждого клиента переопределение пока не делаем — если конкретный
HR-флоу потребует «звонить только в обед» или «больше попыток для VIP-вакансии»,
вернёмся к этому вопросу отдельно.
