# Техническая архитектура + MVP

## Конфигурация VM (Yandex Cloud)

Для старта (до 20 параллельных звонков) хватит:

| Параметр | Значение |
|---|---|
| vCPU | 4 |
| RAM | 8 GB |
| SSD | 100 GB |
| OS | Ubuntu 22.04 LTS |
| Регион | ru-central1-a |
| Ориентировочная цена | 3 500–5 000 ₽/мес |

Запас бюджета использовать так: $50 VM + $100–300 API (STT/LLM/TTS) + $200–400 на маркетинг и пилоты.

## Стек

**Бэкенд:** Python 3.11, FastAPI, Celery + Redis (очереди звонков и фоновые задачи), PostgreSQL 15 (хранение кандидатов, звонков, транскриптов).

**STT (распознавание речи):**
- Основной: **Yandex SpeechKit Streaming** — быстро, дешево (~0,18 ₽/15 сек), хорошо понимает русский.
- Альтернатива: Deepgram Nova-2 (дешевле при больших объёмах, но латентность выше из-за зарубежного хостинга).

**LLM (мозг агента):**
- Основной: **GPT-4o-mini** или **Claude Haiku 4.5** — дёшево, быстро, достаточно для анкет.
- Fallback: **YandexGPT Pro** — если клиент требует, чтобы данные не уходили за границу.

**TTS (синтез голоса):**
- **Yandex SpeechKit** — голоса «Алёна», «Ермил», звучат естественно на русском.
- Альтернатива: ElevenLabs (дороже, но качество выше) — для премиум-клиентов.

**Телефония (SIP/API):**
- **Mango Office** — простой API, широкая распространённость, от 2 500 ₽/мес.
- **Novofon (Zadarma)** — дешевле, API похуже, подойдёт для старта.
- **UIS** — для enterprise, там где важны отчёты.
- Настройка через SIP trunk + медиа-шлюз (FreeSWITCH или готовые webhook-интеграции).

**Фронтенд клиентского кабинета:**
- На старте — **Telegram-бот** вместо веб-кабинета. Серьёзно, это ускоряет запуск на 2–3 недели.
- На 45-й день — простой Next.js/React + Tailwind. Одна таблица звонков + прослушивание записи + экспорт.

**Инфраструктура:**
- Docker Compose (без Kubernetes, переусложнение).
- Nginx как reverse proxy.
- Certbot для Let's Encrypt.
- Sentry для мониторинга ошибок (бесплатный тариф).
- Grafana + Prometheus — позже, когда будет >5 клиентов.

## Схема потока одного звонка

```
Клиент загружает лист кандидатов
    ↓
Cron/worker ставит звонки в очередь (Redis)
    ↓
Celery worker берёт задачу → инициирует звонок через Mango API
    ↓
Кандидат берёт трубку → TTS приветствие + уведомление о записи
    ↓
Цикл диалога:
    Кандидат говорит → Yandex STT (streaming) → текст
    Текст + история → LLM → следующая реплика
    Реплика → Yandex TTS → аудио в звонок
    ↓
После звонка: транскрипт + score + причины → Postgres
    ↓
Если score ≥ порог → уведомление HR в Telegram + карточка в кабинете
Если score < порог → архив + краткая причина
```

## Структура БД (минимум для MVP)

```sql
clients (id, name, tg_chat_id, tariff, active)

```

## MVP-фичи по приоритету

### Must have (первые 30 дней)

1. **Загрузка списка кандидатов** (CSV/Google Sheets link).
2. **Конструктор анкеты через YAML** — без UI, редактируется руками командой.
3. **Автодозвон с 2–3 попытками** при занято/недоступно.
4. **Базовый диалог** по сценарию с ветвлениями.
5. **Уведомление в Telegram** клиенту: «Иван Петров прошёл скрининг, score 8/10, запись → [ссылка]».
6. **Запись звонка + транскрипт** на S3-совместимом Object Storage.

### Should have (дни 31–60)

7. **Веб-кабинет** с таблицей кандидатов и плеером.
8. **Интеграция с hh.ru API** для автоматического импорта откликов.
9. **Экспорт в Excel**.
10. **Несколько вакансий у одного клиента**.
11. **Гибкое расписание обзвона** (только в рабочие часы, учёт часового пояса кандидата).

### Nice to have (дни 61–90)

12. **White-label для кадровых агентств** (свой домен, своё имя бота).
13. **Webhook-интеграция** с CRM клиента (Bitrix24, amoCRM).
14. **Аналитика по вакансии**: какой % кандидатов доходит до конца, где обрываются.
15. **A/B тесты голосов и формулировок**.

## Что НЕ делать в первые 90 дней

- Не писать свой SIP-стек. Только готовые API телефонии.
- Не делать «универсальный конструктор диалогов с визуальным редактором». Это ловушка на 3 месяца разработки.
- Не делать мобильное приложение.
- Не пилить админку для себя в первые 2 месяца. Psql + Telegram-бот достаточно.
- Не писать свой STT/TTS и не деплоить Whisper локально, пока не упрётся в цену API (а упрётся только после 50+ клиентов).

## Ориентировочная стоимость одного звонка (COGS)

Средний скрининг-звонок = 2–3 минуты.

| Компонент | Стоимость за звонок |
|---|---|
| Телефония (2,5 мин) | ~2,5 ₽ |
| STT (Yandex SpeechKit) | ~2 ₽ |
| TTS (Yandex SpeechKit) | ~1,5 ₽ |
| LLM (GPT-4o-mini, ~2k токенов) | ~0,3 ₽ |
| **Итого** | **~6 ₽ за звонок** |

При цене 20–30 ₽ за звонок для клиента маржа 70–80%. Это здоровая юнит-экономика (подробнее в 04_pricing_economics).
