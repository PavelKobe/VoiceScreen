# ROADMAP

Это короткая версия для разработчика. Полный план с продажами/юр/экономикой — в отдельных .md файлах выше по папке (если ты в репозитории, они у тебя локально; если нет — проси у заказчика).

## Текущая веха: **Сквозной тестовый звонок**

Цель: команда `make test-call PHONE=+79990000000 SCENARIO=courier_screening` инициирует реальный звонок, агент проводит анкету, запись и транскрипт сохраняются.

### Что готово (скелет)

- ✅ FastAPI + Celery + SQLAlchemy скелет
- ✅ Mango-клиент (originate, verify signature)
- ✅ Yandex SpeechKit STT/TTS
- ✅ Унифицированный LLM-клиент (OpenAI/Anthropic)
- ✅ Dialog orchestrator с классификатором ответов
- ✅ Scenario-loader (YAML)
- ✅ БД модели (Client, Vacancy, Candidate, Call, CallTurn)
- ✅ Telegram-бот скелет

### Что нужно доделать до первого пилота

1. **Alembic миграции.** Пока нет, модели не накатываются. `alembic init alembic`, затем настроить `env.py` для async.
2. **Mango Media Stream.** Получение аудио-чанков через WebSocket и передача в STT. Сейчас `webhooks.py` только заглушка.
3. **Audio pipeline.** Приклеить TTS-аудио обратно в звонок Mango (нужен Mango media endpoint).
4. **Object Storage.** Загрузка записей в Yandex Object Storage с signed URLs.
5. **Telegram-бот:** реализовать `/register`, загрузку CSV, вывод отчёта по `Client.tg_chat_id`.
6. **`finalize_call` task:** подтянуть запись, посчитать score, отправить в TG.
7. **Таймзоны:** учёт часового пояса кандидата в `schedule_pending_calls`.

## Критичные технические решения

- **Mango vs Novofon.** На MVP — Mango (лучше документация). Если пойдёт — абстракция и добавление Novofon как дешёвой альтернативы.
- **OpenAI vs Anthropic.** На MVP — OpenAI gpt-4o-mini (быстрее, дешевле). Anthropic Claude — для клиентов, которые хотят качество поверх цены.
- **Yandex SpeechKit vs self-hosted Whisper.** Не заморачиваться с self-hosted до 100+ клиентов. Экономия не окупит возню.
